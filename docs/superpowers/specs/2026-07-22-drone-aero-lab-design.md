# drone-aero-lab 设计文档

日期：2026-07-22
状态：已确认，待实现

## 目标

一个纯前端静态项目，用于学习无人机各部件结构与空气动力学，最终发布到 GitHub Pages。

三大类无人机：多旋翼、直升机、垂起固定翼；每类下有子分类（如多旋翼的四轴/六轴/八轴）。用 Three.js 展示每种无人机的主要**结构组成**，并能改变**迎角、材料、顺/逆风、风向**等参数，实时图形化展示**升力与气流**的作用。

核心目标是"展示结构组成"与"空气动力学教学示意"。

## 范围（本次竖切）

本次只实现 **多旋翼** 一条完整竖切：从结构展示到空气动力学全部做完做好，验证整套架构与交互。之后按同一模式扩展直升机、垂起固定翼（各自独立的 spec → plan → 实现循环）。

- 多旋翼子类：**四轴 / 六轴 / 八轴**（差异仅在机臂、电机、螺旋桨的数量与布局，由数据描述）。

明确不做（YAGNI）：
- 真实 CFD 求解；只做教学示意级简化物理。
- 导入外部 GLTF 模型；全部用 Three.js 基础几何体程序化生成。
- 直升机、垂起固定翼（本竖切不含，架构需为其预留数据结构扩展位）。

## 总体架构

数据驱动 + 程序化建模。每种无人机的"部件结构"写成声明式数据（部件名、几何体、变换、材料角色），由通用建造器读取并用 Three.js 拼出带标注的 3D 模型。新增子类/机型只改数据，不改渲染逻辑。

技术栈：Vite + Three.js（OrbitControls）+ 原生 JS 模块 + Vitest（测试空气动力学纯函数）。纯静态。

## 模块划分

每个模块单一职责、接口清晰、可独立理解与测试。

| 模块 | 职责 | 依赖 |
|---|---|---|
| `src/data/drones.js` | 机型/子类/部件的声明式定义（唯一事实源） | 无 |
| `src/scene/` | Three.js 场景、相机、光照、渲染器、resize | three |
| `src/builder/` | 读数据 → 生成带命名的部件 Group；部件高亮 + 标注热点 | scene, data |
| `src/aero/` | 纯函数：由参数算升力/阻力/相对气流/升重比 | 无 |
| `src/viz/` | 气流粒子/流线、升力·重力·风矢量箭头，由 aero 输出驱动 | scene, aero |
| `src/ui/` | 分类选择器 + 滑块（迎角/风速/风向）+ 材料下拉 + 信息面板 | 全部 |
| `src/main.js` | 状态管理、装配、渲染循环 | 全部 |

### 数据结构（`data/drones.js`）

顶层：`categories`（多旋翼/直升机/垂起，本次仅填充多旋翼）→ 每类 `subtypes`（四轴/六轴/八轴）→ 每子类含：
- `parts[]`：`{ id, name, desc, geometry, transform, materialRole }`
  - `geometry`：如 `{ type: 'cylinder', args: [...] }`、`{ type: 'box', args: [...] }`
  - `materialRole`：`structural`（受材料参数影响外观/质感）| `fixed`（固定色，如电池、桨）
- 螺旋桨区分正桨/反桨（CW/CCW），用于结构教学。

## 多旋翼部件清单

机架中心板、机臂、无刷电机、螺旋桨（正/反桨）、飞控 FC、电调 ESC、电池、GPS、脚架。点击部件高亮并显示名称与作用说明。

## 空气动力学模型（教学示意级）

均为简化物理，参数变化实时响应，标注"示意，非精确工程值"。

- **迎角 α**：桨叶剖面升力系数 `CL = min(CL_max, 2π·α)`，超过失速角（约 15°）后 CL 骤降，用于演示失速。
- **升力**：`L = N桨 · ½ · ρ · v² · A · CL(α)`（ρ 空气密度，v 桨叶相对速度，A 桨盘/剖面参考面积）。
- **材料**：碳纤维 / 铝 / 塑料 / 木 → 改变机体密度与总重量 W。实时对比 `L vs W`：
  - `L > W` 爬升，`L ≈ W` 悬停，`L < W` 起不来。
  - 联动机身外观质感；侧栏展示每种材料适用场景（碳纤维→竞速/高性能，铝→耐用性价比，塑料→玩具/入门，木→DIY/原型）。
- **顺/逆风 + 风向**：改变相对气流矢量，可视化气流偏转与需抵消的侧向漂移力。
- **可视化**：绿色↑升力、红色↓重力、蓝色流线气流、风向箭头；数值面板实时刷新。

## 状态与数据流

单一状态对象：`{ category, subtype, params: { aoa, windSpeed, windDir, material }, selectedPartId }`。

流向：UI 改状态 → `aero` 纯函数据状态算出力/流场 → `viz` 更新矢量与流线 → 渲染循环绘制。切换子类时 `builder` 重建模型 Group。

## 测试策略

- `src/aero/` 纯函数用 **Vitest + TDD**：失速点行为、升重比阈值（爬升/悬停/失速）、风向矢量合成、材料→重量映射。
- 结构展示与交互用浏览器截图人工验证。

## 发布

- Vite 配置 `base: '/drone-aero-lab/'`。
- GitHub Actions：push 到 main 时构建并部署到 `gh-pages`。
- README 含本地开发（`npm run dev`）与部署说明。

## 扩展路径

直升机、垂起固定翼沿用同一套 `data + builder + aero + viz` 架构：
- 直升机需扩展主旋翼/尾桨、周期变距等概念的数据与 aero 项。
- 垂起固定翼需引入机翼翼型升力（固定翼段）与旋翼/推进切换的过渡态。
两者各自作为后续独立 spec 处理。
