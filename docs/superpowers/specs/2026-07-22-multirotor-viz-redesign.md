# 多旋翼可视化重塑 设计文档

日期：2026-07-22
状态：已确认，待实现
前置：在多旋翼竖切（`2026-07-22-drone-aero-lab-design.md`）已完成的基础上做增强。

## 背景与动机

首版多旋翼竖切功能完整，但用户反馈了几处必须改进的问题：

1. **UI 设计粗糙**：右侧面板是黑底白字控件罗列，缺分区/层次/图例/设计感；布局不合理；缺少图例说明；控件交互差。
2. **没有坐标轴**：无法判断方向与升力朝向。
3. **迎角语义错误**：多旋翼的"迎角"应是**桨叶迎角**（螺旋桨叶片的安装角/攻角），而非机身迎角。
4. **气流像"下雪"**：随机下落的粒子点读起来像雪，不像气流。
5. **升力只靠文字、且近似 0/1**：应在**模型上图形化线性**展示升力大小的连续变化，而非只有"爬升/失速"文字状态。

## 目标

在现有数据驱动架构（data → builder → aero → viz → ui）上，重塑可视化与 UI，使空气动力学"看得见、连续、可教学"。不改变纯前端静态 + GitHub Pages 的部署方式。

## 范围

仅多旋翼。直升机/垂起仍为后续。本次不改动 `aero.js` 的核心公式（`bladeSpeed=36` 已调好），除非新增按需的纯函数（如每旋翼升力、颜色插值、桨叶剖面矢量），且新增函数继续走 TDD。

明确不做（YAGNI）：真实 CFD；风接入升力物理（侧向漂移力，列为后续）；分类选择器（多旋翼是唯一分类）。

## 关键决策（已与用户确认）

- 升力可视化：**每旋翼升力箭头 + 中心总升力箭头**，长度与颜色随升力**连续**变化；失速时连续变短、变橙/红。
- 迎角：滑块语义为**桨叶迎角 α**；主模型桨叶实际倾转 α；**新增左下角"桨叶剖面"教学图**（α、相对来流、升力三矢量，随滑块联动）。
- 气流：重做为**下洗流线**（上方吸入→经旋翼向下→下洗柱），随风偏斜；不再是随机落点。
- 坐标轴：原点带标注坐标轴（Y=上/升力方向），角落跟随视角的坐标 gizmo。
- UI：分组面板 + 图例 + 控件优化，专业视觉设计（frontend-design 定美术方向）。UI 中文。

## 模块级设计

### 1. 坐标轴与朝向（scene 或新 `viz/axes.js`）

- 原点加坐标轴：X（红）、Y（绿，向上，标注"升力方向↑"）、Z（蓝），轴端用 Sprite 文字标注 `X / Y↑ / Z`。
- 右下角一个小的**视角 gizmo**（跟随主相机旋转的小三轴），帮助判断当前视角朝向。
- 轴含义写入图例。

### 2. 桨叶迎角（`builder.js` + 新 `viz/airfoil.js`）

- **数据/建造**：桨叶网格在建造时记录其"径向轴"（沿机臂方向），`applyBladePitch(meshes, aoaDeg)` 让每片桨绕该径向轴倾转 α，肉眼可见桨叶变斜。
- **剖面图** `viz/airfoil.js`：一个独立的小 Three.js 场景（或 2D canvas，实现取简单者），渲染放大的翼型剖面：
  - 翼型轮廓按 α 倾转；
  - 相对来流箭头（水平指向翼型前缘）；
  - 升力箭头（垂直于来流，长度随 CL(α) 连续变化，失速后变短）；
  - 标注 α 角度值。
  - 挂在页面左下角固定容器，随迎角滑块实时更新。
- 命名统一为"桨叶迎角 α"。

### 3. 下洗气流（重写 `viz.js` 气流部分）

- 为每个旋翼生成一束**下洗流线**：起点在桨盘上方一圈，被吸向桨盘中心，穿过桨盘后加速向下形成下洗柱，在地面附近外扩。
- 用沿流线路径运动的粒子或短线段表现"流动"，密度适中、成束，读感是气流不是雪。
- 风矢量对流线做整体平流偏斜（下洗柱随风倾斜/拖尾）。
- 下洗强度（速度/箭头）随总升力连续变化。

### 4. 升力可视化（`aero.js` 新增纯函数 + 重写 `viz.js` 箭头部分）

- **新增纯函数（TDD）**：
  - `perRotorLift({ bladeSpeed, refArea, aoaDeg, airDensity })` → 单旋翼升力（`computeLift` 除以 rotorCount 的等价单体，或独立实现），供每旋翼箭头。
  - `liftColor(lift, weight)` → 颜色（连续插值）：按升重裕度 `r=lift/weight` 从红(r≤~0.8)→橙→绿(r≥~1.2)平滑过渡，返回 `{r,g,b}` 或 hex。**连续**，非阈值跳变。
- **每旋翼升力箭头**：每个电机/桨上方一支向上箭头，长度∝该旋翼升力、颜色=liftColor。
- **中心总升力箭头**：粗箭头，长度∝总升力；红色重力箭头长度∝重量，并排对比。
- 文字读数保留为辅助（仍标"示意值"）。
- 失速表现为箭头连续变短 + 颜色连续转橙/红。

### 5. UI 重设计（`ui.js` + `index.html`/样式）

- **分组**：① 机型（分类只读"多旋翼"+子类下拉）② 飞行参数（桨叶迎角 α / 风速 / 风向 / 材料）③ 实时读数（总升力、重量、每旋翼升力、升重比条）④ 部件说明。
- **图例区**：箭头颜色（绿=升力、红=重力、青=风、蓝=下洗气流）、坐标轴（Y↑=升力方向）、失速色变含义。
- **控件**：滑块带当前值+单位就近显示；选中部件时部件列表对应项高亮；升重比用一个连续的水平条/仪表表现而非纯文字。
- **视觉**：统一配色、层次、卡片、间距、排版；用 frontend-design 技能定方向。中文。
- 布局仍为右侧面板 + 左下角剖面图 + 右下角 gizmo，主 3D 占主区域。

## 状态与数据流（增量）

`state` 增加不了太多——参数不变（aoaDeg 现语义为桨叶迎角）。数据流：UI 改状态 → aero 算总升力/每旋翼升力/颜色 → viz 更新每旋翼箭头/总箭头/下洗流线 + airfoil 剖面 + builder 桨叶倾转 → 渲染。

## 测试策略

- 新增 aero 纯函数（`perRotorLift`、`liftColor`）用 **Vitest + TDD**（每旋翼升力与总升力一致性、颜色随裕度单调/连续、边界不跳变）。
- 可视化/UI/剖面/流线用浏览器截图人工验证（控制器在里程碑做）。
